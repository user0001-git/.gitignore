Apa itu .gitignore?

.gitignore adalah file yang digunakan dalam proyek Git untuk menentukan file atau direktori yang harus diabaikan oleh Git. Dengan menggunakan file ini, Anda dapat memastikan bahwa file tertentu, seperti file konfigurasi lokal, file sementara, atau file build, tidak akan diunggah ke repository Git.

"Sebuah file yang bisa kita simpan di dalam repository git kita supaya kedepannya pada saat melakukan add dan commit, file tidak terbawa ke dalam add." (Sandhika Galih)

Cara Menggunakan .gitignore

  1. Buat file bernama .gitignore di direktori root proyek Anda.
  2. Tulis nama file atau pola file yang ingin diabaikan. Contoh: 
    # Mengabaikan file sementara
*.tmp

# Mengabaikan direktori build
/build/

# Mengabaikan file konfigurasi lokal
config.json
  3. Simpan file .gitignore dan pastikan file tersebut sudah ditambahkan ke repository Anda.
  4. Gunakan perintah git status untuk memastikan bahwa file yang terdaftar di .gitignore tidak muncul di daftar perubahan.

Untuk mempermudah, Anda bisa menggunakan layanan seperti gitignore.io. Layanan ini memungkinkan Anda untuk membuat file .gitignore yang sesuai dengan kebutuhan proyek Anda, cukup dengan memilih jenis teknologi atau framework yang Anda gunakan.
